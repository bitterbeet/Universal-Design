What is Universal Design?

Universal design blends accessibility, inclusive language/content, and resilient code. It means designing so the most people possible can perceive, operate, understand, and interact with the interface.

Principles (short)

  • Perceivable: information must be presented in ways people can perceive (text alternatives, captions, good contrast).
  • Operable: interface controls must be usable by keyboard and assistive tech.
  • Understandable: content and UI behavior should be predictable & clear.
  • Robust: work across devices, agents, and years of tech (semantic markup, ARIA when needed).

Concrete examples

Design

Use scalable type (rem units), contrast tokens, flexible layouts, and motion reduction options. Avoid color-only UI cues.

Content

Write clear headings, short paragraphs, link context, and use descriptive alt text for images.

Code

Prefer semantic elements: <header>, <nav>, <main>, <button>. Use ARIA only to enhance semantics that HTML cannot express.

Tiny code snippet (semantic & accessible)

<button aria-pressed="false">Like</button>
